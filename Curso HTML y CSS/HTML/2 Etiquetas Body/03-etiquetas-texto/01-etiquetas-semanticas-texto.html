1 Fragmentos de texto
<a> Fragmento de texto que enlaza a otra página diferente. Buena práctica
<strong> Fragmento de texto importante o palabras clave. Buena práctica
<em> Fragmento de texto enfatizado respecto a la frase que lo contiene. Buena práctica
<mark> Fragmento de texto resaltado, como referencia o anotación, debido a su relevancia o importancia en un contexto particular. Buena práctica -> No confundir con la etiqueta <strong>. La etiqueta <mark> se usa para denotar contenido con cierto grado de relevancia, mientras que <strong> se usa para dar importancia a una parte del texto
<b> Fragmento de texto sin importancia destacable (fines utilitarios). Mala práctica según el W3C y MDN, es lo mismo que poner en CSS: "font-weight: bold;". Solo se puede utilizar cuando no hay un elemento semántico apropiado para sustituir a esta etiqueta. Normalmente se usa <strong> en vez de <b>
<i> Fragmento de texto con voz o tono alternativo al resto. Mala práctica según el W3C y MDN, es lo mismo que poner en CSS: "font-style: italic;". Solo se puede utilizar cuando no hay un elemento semántico apropiado para sustituir a esta etiqueta. Normalmente se usa <em> en vez de <i>
<u> Fragmento de texto para nombres propios o escritura incorrecta intencionada. Mala práctica según el W3C y MDN, es lo mismo que poner en CSS: "text-decoration: underline;". Solo se puede utilizar cuando hacemos anotaciones de errores gramaticales o un caso muy específico. Normalmente se usa <em> en vez de <u>
<s> Fragmento de texto inexacto o que ya no es relevante. Buena práctica
<span> Fragmento de texto genérico sin significado (útil para seleccionar). Buena práctica
2 Fragmentos con significado
<sup> Superíndice Buena práctica
<sub> Subíndice Buena práctica
<small> Anotaciones menores Buena práctica
<cite> Fragmento de texto con el título de un trabajo creativo: obras, libros... Buena práctica -> Está bastante estendido el mal uso de este elemento, mucha gente lo usa para marcar el texto de las citas en vez del autor, para marcar el texto están los elementos q (en línea) y blockquote (en bloque)
<q> Cita o frase extraída de otro contexto. Buena práctica
<abbr> Abreviatura o acrónimo. Buena práctica
<dfn> Definición: término que posteriormente será definido. Buena práctica
3 Fragmentos técnicos/informáticos
<kbd> Entrada de información del usuario (generalmente, combinación de teclado). Buena práctica
<samp> Salida de información de un programa informático. Buena práctica
<var> Variable (contexto matemático o informático). Buena práctica
<time> Indica una fecha/hora legible para humanos, con formato para máquinas opcional. Buena práctica
<data> Información equivalente orientado a máquinas. Buena práctica
<code> Fragmento de código fuente (en línea). Buena práctica
4 Fragmentos de edición
<ins> Fragmento de texto o información añadida a posteriori de la publicación inicial. Buena práctica
<del> Fragmento de texto o información eliminada a posteriori de la publicación inicial. Buena práctica
